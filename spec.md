# 나만의 일기장 (My Diary) — 기능 명세서

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 이름 | 나만의 일기장 (My Diary) |
| 목적 | 개인 일기를 작성·관리할 수 있는 웹 앱 |
| 대상 사용자 | 매일의 생각과 감정을 기록하고 싶은 개인 사용자 |

---

| 영역 | 선택 |
|------|------|
| 프레임워크·라우팅 | Next.js — **App Router** (`app/` 디렉터리) |

## 3. 기능 목록 (MVP)

### 3-1. 인증

- **설명**: Supabase Auth를 활용한 이메일+비밀번호 기반 회원가입, 로그인, 로그아웃
- **페이지**:
  - `/login` — 로그인 폼
  - `/signup` — 회원가입 폼
- **동작**:
  - 회원가입 시 이메일·비밀번호를 입력하면 Supabase Auth에 사용자 생성
  - 로그인 성공 시 일기 목록(`/diaries`)으로 리다이렉트
  - 로그아웃 시 세션 제거 후 로그인 페이지로 이동
  - 비로그인 상태에서 보호 페이지 접근 시 `/login`으로 리다이렉트

### 3-2. 일기 작성

- **설명**: 제목, 본문, 날짜, 오늘의 기분을 입력하고 저장
- **페이지**: `/diaries/new`
- **동작**:
  - 제목(필수), 본문(필수), 날짜(기본값: 오늘), 기분(필수) 입력
  - 저장 시 `diaries` 테이블에 INSERT
  - 저장 완료 후 상세 페이지(`/diaries/[id]`)로 이동

### 3-3. 일기 목록

- **설명**: 로그인한 사용자의 일기를 최신순으로 조회
- **페이지**: `/diaries`
- **동작**:
  - `diary_date` 기준 내림차순 정렬
  - 각 항목에 제목, 날짜, 기분 아이콘 표시
  - 항목 클릭 시 상세 페이지로 이동
  - 기분별 필터링 지원 (3-6 참고)

### 3-4. 일기 상세 보기

- **설명**: 선택한 일기의 전체 내용을 표시
- **페이지**: `/diaries/[id]`
- **동작**:
  - 제목, 본문, 날짜, 기분 표시
  - 수정·삭제 버튼 제공
  - 본인 일기가 아니거나 존재하지 않으면 404

### 3-5. 일기 수정 / 삭제

- **설명**: 기존 일기의 내용을 편집하거나 삭제
- **페이지**: `/diaries/[id]/edit` (수정)
- **동작**:
  - 수정: 기존 데이터를 폼에 채워 놓고 변경 후 저장 (UPDATE)
  - 삭제: 확인 다이얼로그 표시 후 DELETE, 목록으로 이동

### 3-6. 기분(감정) 기록

- **설명**: 일기 작성 시 오늘의 기분을 선택하고, 목록에서 기분별 필터링
- **기분 종류**:
  | 값 | 라벨 |
  |------|------|
  | `happy` | 행복 |
  | `sad` | 슬픔 |
  | `angry` | 화남 |
  | `anxious` | 불안 |
  | `neutral` | 보통 |
- **동작**:
  - 작성/수정 폼에서 기분 선택 UI 제공
  - 일기 목록에서 기분 필터 드롭다운 또는 버튼으로 필터링
  - 선택된 기분에 해당하는 일기만 표시 (`mood` 컬럼 WHERE 조건)

## 4. Supabase 테이블 스키마

### `diaries` 테이블

| 컬럼 | 타입 | 제약 조건 | 설명 |
|------|------|-----------|------|
| `id` | `uuid` | PK, default `gen_random_uuid()` | 일기 고유 식별자 |
| `user_id` | `uuid` | NOT NULL, FK → `auth.users.id` ON DELETE CASCADE | 작성자 |
| `title` | `text` | NOT NULL | 제목 |
| `content` | `text` | NOT NULL | 본문 |
| `diary_date` | `date` | NOT NULL | 일기 날짜 |
| `mood` | `text` | NOT NULL, CHECK (`mood` IN ('happy','sad','angry','anxious','neutral')) | 기분 |
| `created_at` | `timestamptz` | default `now()` | 생성 시각 |
| `updated_at` | `timestamptz` | default `now()` | 수정 시각 |

> `updated_at`은 UPDATE 시 자동 갱신되도록 트리거 또는 애플리케이션 레벨에서 처리한다.

## 5. 페이지 구조 (라우팅 맵)

```
app/
├── (auth)/
│   ├── login/
│   │   └── page.tsx          ← 로그인
│   └── signup/
│       └── page.tsx          ← 회원가입
├── (main)/
│   └── diaries/
│       ├── page.tsx          ← 일기 목록 + 기분 필터
│       ├── new/
│       │   └── page.tsx      ← 일기 작성
│       └── [id]/
│           ├── page.tsx      ← 일기 상세
│           └── edit/
│               └── page.tsx  ← 일기 수정
├── layout.tsx                ← 루트 레이아웃
└── page.tsx                  ← 랜딩 (로그인 여부에 따라 리다이렉트)
```

- `(auth)` 그룹: 비로그인 사용자용 레이아웃
- `(main)` 그룹: 로그인 사용자용 레이아웃 (네비게이션 바, 로그아웃 버튼 포함)
